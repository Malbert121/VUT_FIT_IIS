Implementace

Frameworks: .NET8(ASP.NET, EntityFrameworkCore), Node.js(React+TypeScript)

1. Frontend:

Location: /conventus-client
Description: This part of the project implements the application's user interface using React.

Main Folders and Files:

  • Dockerfile: Docker file for client side.
  • ngnix.conf: Configurations for HTTP server NGNIX for client side.
  • src/Components: Contains user interface components that make up the various parts of the application.
  • src/context: Includes the implementation of state management (e.g., using React Context API).
  • src/Pages: Contains pages representing different sections of the application.
  • src/Routes: Implements routing between the application's pages.
  • api.tsx: Contains the API client for communication with the backend.
  • App.tsx: The main file for initializing the frontend application.

2. Backend API: Conventus.API

Location: Conventus.API
Description: This part of the project provides a RESTful API for data processing and handling client requests.

Main Folders and Files:

  • Dockerfile: Docker file for server side.
  • Controllers: Contains API controllers that implement various endpoints (e.g., CRUD operations).
  • appsettings.json and appsettings.Development.json: Configurations for different environments (e.g., database connection, API settings).
  • Program.cs: The main file for configuring and starting the backend server.
  • launchSettings.json: Settings for launching the application (e.g., port, environment).

3. Data Layer: Conventus.DAL

Location: Conventus.DAL
Description: Contains the application's data logic, including database access implementation.

Main Folders and Files:

  • EfStructures: Implements DbContext and database connection initialization.
    • Key File: ConventusDbContext.cs (FluentApi).
    • Key File: ConventusDbContextFactory.cs (used for setting up the connection string).
  • Initialization: Contains initialization logic or database seeding.
  • Migrations: Folder for managing database migrations.
  • Repositories: Implements repositories for working with entities.

4. Models: Conventus.Models

Location: Conventus.Models
Description: Contains entity models that correspond to the structure of database tables or objects used in the API.

5. Other Important Files

  • init.sql: Autogenerated script by EntityFrameworkCore from migrations for initialization database, schemes and testing data
  • doc.html: Documintation
  • docker-compose.yml: Configuration file for docker-compose of project.
  • Dockerfile.sqlserver: Docker file for MSSQL server.
  • Dockerfile.sqlserver_init: Docker file for sqlcmd for MSSQL server (Need for first initialization DB and test data).
  • Conventus.sln: Main solution file for opening the project in Visual Studio.
  • db_diagram.png: Database schema for a visual overview of the structure.

Installation

Installation from Docker

  1. Need installed docker and docker-compose. After all console command will make from root of project
  2. If you need stop(and removes) containers: 
    docker-compose down
  3. If you need stop(and removes) containers and clean volumes of db: 
    docker-compose down -v
  4. Build docker: 
    docker-compose build
  5. Start MSSQL, application server, client (WARNING it need time for full running, wait please): 
    docker-compose up sqlserver server client
  6. If you need create/recreate db and test data (WARNING Up after when you have sqlserver full bilded and runned, HAVE 2 MINUTS sleeping for connection): 
    docker-compose up sqlserver-init
  7. Your application is availeble from: localhost:3000/

Installation from source code

  1. Navigate to the file located at: Conventus.DAL\EfStructures\ConventusDbContextFactory.cs.
  2. Locate the line var connectionString and set it to the correct connection string for your database.
  3. Open the Conventus.API folder.
  4. Ensure the following sections in appsettings.json are correctly configured:
    • Database Connection: Update the ConnectionStrings section with the database connection string.
  5. Open the project in your IDE by double-clicking on the Conventus.sln file.
  6. Apply any necessary database migrations to set up the database(console must be opened in Conventus.DAL): 
    
dotnet ef migrations add InitialCreate
dotnet ef database update
  7. Ensure the database is running and accessible with the configured connection string.
  8. Build and run the project(Conventus.API):
    • In Visual Studio, press F5 to start the API server.
    • If using a different IDE or terminal, navigate to the Conventus.API folder and run: 
      dotnet run
  9. Your application is availeble from: localhost:3000/
